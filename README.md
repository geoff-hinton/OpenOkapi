这是一个用于仿真和可视化“基于电网电流斜率的储能动态控制”的Python脚本。因为它画出的图形很像Okapi（一种非洲动物，身体线条流畅，腿部有斑马条纹），这可能是对图中曲线（负载电流、储能电流、电网电流及其斜率）交互形态的生动比喻，多色曲线簇与斜率的阈值线交织，或许会让人联想到Okapi独特的斑纹图案。

详细解释脚本的功能、核心逻辑、参数含义以及输出内容：

一、脚本目标
模拟一个用户侧储能系统在检测到电网电流（I_grid）斜率超过设定阈值时，自动进入“跟随模式”，以比例系数 kk 补偿负载电流超出背景电流（Inormal=5AInormal=5A）的部分；当斜率回落至阈值以下时，储能切换到“保持模式”，锁定当前输出电流不变。最终生成数据表格和对比图，用于分析储能能否有效削减冲击电流、降低需量电费。

二、核心算法逻辑
1. 输入数据（固定）
• 时间轴 t：10~15 ms，步长约 0.0476 ms（对应采样频率 21 kHz）。
• 负载电流 I_load：模拟中央空调启动过程，从 5A 背景电流线性上升到 605A（峰值在 15 ms）。
2. 动态控制规则
脚本逐个时间步循环计算：
1. 计算电网电流斜率
    dI_grid[i] = (I_grid[i] - I_grid[i-1]) / dt
2. 模式判断（滞后一步）
    ◦ 若上一时刻的 dI_grid[i-1] ≥ Dref → 进入 跟随模式
    ◦ 否则 → 进入 保持模式
3. 目标电流计算
    ◦ 跟随模式：
        target = min(I_max, max(0, k * (I_load[i] - I_normal)))
        即补偿超出背景电流的部分，按系数 k 缩放，并限幅在 I_max 以内。
    ◦ 保持模式：
        target = bat_current（即当前实际储能电流，保持不变）
4. 实际电流平滑
    实际电流 bat_current 以最大变化率 di_dt_max 向 target 逼近，每步最大改变 delta_I_max = di_dt_max * dt。
5. 更新电网电流
    I_grid[i] = I_load[i] - bat_current

三、输出内容
1. 表格（打印到终端或保存为文件）
包含列：
t(ms)、I_load(A)、I_bat_target(A)、I_bat_actual(A)、I_grid(A)、dI_grid(A/s)、Mode（跟随/保持）
2. 图片（自动保存为 Figure4_k*_Imax*_didt*_Dref*_dynamic.png）
• 主纵轴：负载电流（蓝色虚线）、储能电流（绿色点划线）、电网电流（红色实线）
• 副纵轴：电网电流斜率 dI_grid（紫色点线），以及触发阈值水平线（橙色虚线）
• 竖向虚线：标记每一个从“保持”切换到“跟随”的时刻
• 黄色阴影：储能补偿的能量区域（I_load > I_grid 的部分）

四、为什么图形像 Okapi？
你可能注意到 绿色储能电流 呈现阶梯状爬升（跟随模式时上升，保持模式时平台），而 红色电网电流 的斜率（红色线）在阈值线附近反复穿越——这种斑马条纹般的交错，加上整体曲线流畅的轮廓，恰似 Okapi 臀部和大腿上的横向条纹。
用“okapi”命名项目，既能体现动态控制的“条纹切换”特征，又足够幽默和易记。

五、脚本的局限与改进方向
• 滞后判断：模式切换依赖上一时刻的斜率，会有一个时间步的延迟（~0.0476 ms）。可改为实时预测或更复杂的滤波。
• 保持模式下的目标电流：设定为 bat_current，导致储能电流不再跟随负载变化，适用于“冲击过后不再补刀”的场景；若需要缓慢泄放，可增加一个缓慢衰减项。
• 未模拟放电结束后的充电恢复：脚本只涵盖冲击上升段（~15 ms），实际储能需在谷段恢复 SOC。
